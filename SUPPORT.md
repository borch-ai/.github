# Support

## Getting Help

### Bug Reports & Feature Requests

Open an **Issue** in the relevant repository using the provided issue templates:

| Repository | Issues |
|---|---|
| [Kiln](https://github.com/borch-ai/kiln/issues) | Business orchestration, Foundry, CLI |
| [Pithos](https://github.com/borch-ai/pithos/issues) | Book factory pipeline |
| [Powerword](https://github.com/borch-ai/powerword/issues) | MCP plugins, Critic, telemetry |
| [Aeolian](https://github.com/borch-ai/aeolian/issues) | Music & media pipeline |
| [Lighthouse](https://github.com/borch-ai/lighthouse/issues) | Telemetry sink & analytics |
| [Lamplighter](https://github.com/borch-ai/lamplighter/issues) | Mobile monitoring |

### Checking In-Progress Work

Before opening an issue, check whether the problem is already tracked:

- **`ROADMAP.md`** in each repo — lists all planned and completed tasks by phase
- **`plans/`** directory — individual implementation plans with detailed scope

### Security Issues

**Do not open a public issue for security vulnerabilities.** Email **security@sinope.press** instead. See [SECURITY.md](SECURITY.md) for our responsible disclosure policy.

### Ecosystem Overview

For a high-level map of how the components fit together, see the [borch-ai organization profile](https://github.com/borch-ai).
