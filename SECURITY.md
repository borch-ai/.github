# Security Policy

## Supported Versions

All repositories in the **borch-ai** organization follow a rolling-release model. Only the latest version of each component is actively maintained.

## Reporting a Vulnerability

We take the security of the borch-ai ecosystem seriously. If you discover a security vulnerability in any of our repositories, please **do not open a public GitHub issue**.

Report vulnerabilities responsibly by emailing:

**security@sinope.press**

### What to Include

To help us investigate and patch quickly, please provide:

- A clear description of the vulnerability
- Steps to reproduce (including a proof-of-concept if possible)
- The affected repository and version/commit
- The potential impact (data exposure, privilege escalation, etc.)

### Our Commitment

Upon receiving a vulnerability report, we commit to:

1. **Acknowledging receipt within 48 hours**
2. Investigating and working on a resolution diligently
3. Providing progress updates as we work toward a fix
4. Publicly disclosing the vulnerability and crediting your contribution once a patch is deployed (unless you prefer to remain anonymous)

We will not take legal action against researchers who report vulnerabilities in good faith and adhere to this policy.
